# TN Scheme Navigator AI  
## AI-Powered Tamil Nadu Government Scheme Assistant with Hybrid Graph RAG

TN Scheme Navigator AI is a production-style AI application that helps users search, understand, and verify Tamil Nadu Government scheme information.

The application combines **Vector RAG**, **Neo4j Knowledge Graph RAG**, **Hybrid AI Routing**, and **Tamil / English / Bilingual response support** to provide citizen-friendly answers with official source references.

---

## Project Objective

The objective of this project is to build a reliable AI assistant that can:

- Search Tamil Nadu Government scheme information
- Explain schemes in simple citizen-friendly language
- Support English, Tamil, and Bilingual responses
- Retrieve relevant scheme information using ChromaDB Vector RAG
- Understand scheme relationships using Neo4j Knowledge Graph RAG
- Combine semantic search and graph reasoning using Hybrid RAG
- Show official source references for verification
- Provide a clean Streamlit interface suitable for demo and submission

## Active AI Engine Flow

The Streamlit app uses the following active engine flow:

```text
app.py
  ↓
run_hybrid_ai()
  ↓
hybrid_rag_engine.ask_hybrid_ai()
  ↓
├── rag_engine.ask_scheme_ai()
│     └── ChromaDB Vector RAG
│
└── graph_rag_engine.ask_graph_ai()
      └── Neo4j AuraDB Graph RAG
```

If `hybrid_rag_engine.py` is unavailable or has an import issue, the app safely falls back to:

```text
rag_engine.ask_scheme_ai()
```

This ensures the app can still run in Vector RAG mode even if Graph RAG is not available.

## How the App Works

### 1. Data Collection

The app scrapes Tamil Nadu Government scheme data and stores it as a structured CSV file.

Output:

```text
data/tn_scheme_details.csv
```

---

### 2. Vector RAG Layer

The CSV data is converted into LangChain documents.

OpenAI embeddings are generated and stored in ChromaDB.

This helps answer meaning-based questions.

Example:

```text
What schemes are available for farmers?
```

Vector RAG is useful when the user asks broad or semantic questions.

---

### 3. Knowledge Graph Layer

The same scheme data is loaded into Neo4j AuraDB as a graph.

Graph nodes include:

```text
Scheme
Department
Beneficiary
Benefit Type
Sponsor
Keyword
```

Graph relationships include:

```text
Scheme → BELONGS_TO → Department
Scheme → FOR_BENEFICIARY → Beneficiary
Scheme → PROVIDES_BENEFIT → Benefit Type
Scheme → SPONSORED_BY → Sponsor
Scheme → HAS_KEYWORD → Keyword
```

Graph RAG is useful for relationship-based questions.

Example:

```text
Which schemes are sponsored by State?
```

---

### 4. Hybrid RAG Layer

Hybrid RAG combines both retrieval approaches.

```text
ChromaDB Vector RAG
+
Neo4j Knowledge Graph RAG
+
OpenAI Final Response
```

This provides richer answers by using both semantic similarity and structured graph relationships.

---

## Prompt Quality and Tamil Response Instruction

The app uses explicit language instructions to control the final AI answer.

### Tamil Response Instruction

```text
Respond fully in Tamil.
Use simple, clear Tamil that a normal citizen can understand.
Keep government scheme names in English if exact translation may confuse the user.
Explain eligibility, benefit, how to apply, and source clearly.
```

### English Response Instruction

```text
Respond in clear professional English.
Keep the answer simple, structured, and citizen-friendly.
```

### Bilingual Response Instruction

```text
Respond in both English and Tamil.
First give a short English answer.
Then provide the Tamil explanation below it.
Keep the Tamil simple and citizen-friendly.
```

### Final Hybrid RAG Prompt Structure

The final answer generation prompt is structured as:

```text
You are TN Scheme Navigator AI, an expert assistant for Tamil Nadu Government schemes.

User question:
{question}

Selected response language:
{language}

Language instruction:
{language_instruction}

Routing decision:
{route}

Vector RAG answer:
{vector_answer}

Graph RAG answer:
{graph_answer}

Official source references:
{source_text}

Create a final high-quality answer.

Rules:
1. Answer only using the provided Vector RAG and Graph RAG information.
2. Do not invent scheme details.
3. If information is not available, say clearly that the available data does not confirm it.
4. Mention scheme names clearly.
5. Explain:
   - Suitable schemes
   - Beneficiaries
   - Benefits
   - Sponsor or department if available
   - How to apply if available
6. Keep the answer simple and citizen-friendly.
7. Add a short note reminding the user to verify from the official source link.
```

## Neo4j Browser Queries

After loading graph data into Neo4j AuraDB, use these Cypher queries in Neo4j Browser.

### View graph

```cypher
MATCH (n) RETURN n LIMIT 50;
```

### View scheme relationships

```cypher
MATCH (s:Scheme)-[r]->(n)
RETURN s.name, type(r), n.name
LIMIT 50;
```

### State sponsored schemes

```cypher
MATCH (s:Scheme)-[:SPONSORED_BY]->(sp:Sponsor)
WHERE toLower(sp.name) CONTAINS "state"
RETURN s.name AS Scheme, sp.name AS Sponsored_By
LIMIT 25;
```

### Farmer schemes

```cypher
MATCH (s:Scheme)-[:FOR_BENEFICIARY]->(b:Beneficiary)
WHERE toLower(b.name) CONTAINS "farmer"
RETURN s.name AS Scheme, b.name AS Beneficiary
LIMIT 25;
```

### Grant schemes

```cypher
MATCH (s:Scheme)-[:PROVIDES_BENEFIT]->(bt:BenefitType)
WHERE toLower(bt.name) CONTAINS "grant"
RETURN s.name AS Scheme, bt.name AS Benefit
LIMIT 25;
```
